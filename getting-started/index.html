    Mengukur Jarak Data¶

    A. Mengukur Jarak Tipe Data Numerik¶

    1.Minkowski Distance¶

    Pengertian : Jarak Minkowski merupakan metrik dalam ruang vektor yang dapat dikatakan sebagai generalisasi dari Euclidean distance dan Manhattan distance. Jarak Minkowski dapat dinyatakan dalam formula berikut: $$ d _ { \operatorname { min } } = ( sum _ { i = 1 } ^ { n } | x _ { i } - y _ { i } | ^ { m } ) ^ { \frac { 1 } { m } } , m \geq 1 $$ Keterangan:

    • n = banyak fitur

    • i = batas jumlah data ke-i

    • m = banyaknya data

    • xi = nilai x pada data ke-i

    • yi = nilai y pada data ke-i

    2.Manhattan Distance¶

    Manhattan distance adalah salah satu pengukuran yang paling banyak digunakan meliputi penggantian perbedaan kuadrat dengan menjumlahkan perbedaan absolute antar variable, dimana a dan b adalah koordinat titik atau vektor kedua dokumen. Prosedur ini disebut blok absolute atau lebih dikenal dengan city block distance.Manhattan distance dapat didefinisikan dengan menggunakan formula berikut: $$ d _ { \operatorname { man } } = \sum _ { i = 1 } ^ { n } \left| x _ { i } - y _ { i } \right| $$ Keterangan:

    • n = banyak fitur

    • i = batas jumlah data ke-i

    • m = banyaknya data

    • x dan y = sebuah titik yang akan diukur jaraknya

    3.Euclidean Distance¶

    Pengertian : Euclidean distance adalah perhitungan jarak (kesamaan) dua buah titik dalam euclidean space. Metode euclidean ini bisa kita gunakan untuk perhitungan jarak terdekat, berikut formula menggunakan metode tsb:

    • i = batas jumlah data ke-i

    • yij = nilai y pada training data
    4.Average Distance¶

    Pengertian: Average distance merupakan salah satu bentuk pemodifikasian dari perhitungan Euclidean distance untuk memperbaiki hasil yang diperoleh, dua titik x,y merupakan titik dalam sebuah ruang dimensi -n. Average distance dapat didefinisikan dalam formula berikut ini: $$ d _ { a v e } = \left ( \frac { 1 } { n } \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } \right) ^ { \frac { 1 } { 2 } } $$ Keterangan:

    • n = banyak fitur

    • i = batas jumlah data ke-i

    • m = banyaknya data

    • xik = nilai x pada training data

    • yij = nilai y pada training data

    5.Weighted Euclidean Distance¶

    ​ Weighted euclidean distance merupakan modifikasi lain dari euclidean distance untuk meminimalkan hasil yang diperoleh agar lebih sesuai dengan menggunakan formula berikut ini: $$ d _ { w e } = \left ( \sum _ { i = 1 } ^ { n } w _ { i } ( x _ { i } - y _ { i } \right) ^ { 2 } ) ^ { \frac { 1 } { 2 } } $$

    Keterangan:

    • xi = nilai x pada data ke-i
    • yi = nilai y pada data ke-i
    • n = batas jumlah banyaknya data
    • i =1 = batas data dari i = 1
    • wi= bobot pada atribut ke-1
    6.Chord Distance¶

    ​ Chord distance adalah salah satu ukuran jarak modifikasi Euclidean distance untuk memperbaiki kekurangan/kelemahan dari Euclidean distance agar hasil yang didapat lebih optimal dan sesuai dengan yang diinginkan. perhitungan chord distance ini dapat dipecahkan juga dengan menggunakan skala pengukuran yang baik dan dapat dihitung dari data yang tidak dinormalisasi . Chord distance didefinisikan dengan formula berikut: $$ d _ { \text {chord} } = \left ( 2 - 2 \frac { \sum _ { i = 1 } ^ { n } x _ { i } y _ { i } } { | x | _ { 2 } | y | _ { 2 } } \right) ^ { \frac { 1 } { 2 } } $$

    Keterangan:

    • xi = nilai x pada data ke-i
    • yi = nilai y pada data ke-i
    • n= batas jumlah banyaknya data
    7.Mahalanobis Distance¶

    ​ Mahalanobis Distance digunakan dalam Digital Image Processing untuk mengidentifikasi nilai variant selama pengklasifikasian gambar. Mahalanobis distance adalah suatu metode statistika yang digunakan untuk mendapatkan suatu data dengan jarak tertentu terhadap mean data tersebut sehingga diperoleh suatu penyebaran data yang memiliki pola terhadap nilai mean. Berikut adalah pedefinisian dari mahalanobis distance: $$ d _ { m a h } = \sqrt { ( x - y ) S ^ { - 1 } ( x - y ) ^ { T } } $$

    • S^-1= matrik identitas dari data yang diketahui
    • (x-y)^T= transpose dari selisih metric x dan y
    8.Cosine Measure¶

    Ukuran Cosine similarity lebih banyak digunakan dalam similaritas dokumen dan dinyatakan dengan formula berikut ini: $$ Cosine(x,y)=\frac { \sum _ { i = 1 } ^ { n } x _ { i } y _ { i } } { | x | _ { 2 } | y | _ { 2 } } $$

    • xi = nilai x pada data ke-i
    • yi = nilai y pada data ke-i
    • n= batas jumlah banyaknya data
    9.Pearson Correlation¶

    ​ Korelasi Pearson merupakan salah satu ukuran korelasi yang digunakan untuk mengukur kekuatan dan arah hubungan linier dari dua veriabel. Dua variabel dikatakan berkorelasi apabila perubahan salah satu variabel disertai dengan perubahan variabel lainnya, baik dalam arah yang sama ataupun arah yang sebaliknya. Harus diingat bahwa nilai koefisien korelasi yang kecil (tidak signifikan) bukan berarti kedua variabel tersebut tidak saling berhubungan. Mungkin saja dua variabel mempunyai keeratan hubungan yang kuat namun nilai koefisien korelasinya mendekati nol, misalnya pada kasus hubungan non linier. Dengan demikian, koefisien korelasi hanya mengukur kekuatan hubungan linier dan tidak pada hubungan non linier. Harus diingat pula bahwa adanya hubungan linier yang kuat di antara variabel tidak selalu berarti ada hubungan kausalitas, sebab-akibat. Kolerasi pearson dapat didefinisikan dalam formula berikut: $$ Pearson ( x , y ) = \frac { \sum _ { i = 1 } ^ { n } ( x _ { i } - \mu _ { x } ) ( y _ { i } - \mu _ { y } ) } { \sqrt { \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } } \sqrt { \sum _ { i = 1 } ^ { n } ( x _ { i } - y _ { i } ) ^ { 2 } } } $$

    Keterangan:

    • xi= nilai x pada data ke-i
    • yi= nilai y pada data ke-i
    • n= batas jumlah banyaknya data
    • i=1 = batas data dari i = 1
    • μx = nilai μ pada data ke-x
    • μy= nilai μ pada data ke-y

    B. Mengukur Jarak Atribut Binary¶

    ​ Binary attribute adalah nominal atribut yang hanya memiliki dua kategori atau keadaan yaitu 0 dan 1. 0 berarti tidak ada dan 1 berarti ada. Binary attribute biasanya diartikan sebagai Boolean jika kedua keadaannya adalah true(benar) dan false(salah). Binary attribute bisa simetris(symmetric) dan bisa asimetris(assymmetric). Simetris jika kedua nilainya bernilai sama/setimbang harganya, sehingga tidak bisa diberi kode 0 atau 1, sedangkan asimetris kedua nilainya tidak setimbang harganya, sehingga dapat diberi kode 0 atau 1.

    • Untuk atribut biner simetris: $$ d ( i , j ) = \frac { r + s } { q + r + s + t } $$

    • Untuk atribut biner asimetris: $$ d ( i , j ) = \frac { r + s } { q + r + s } $$

    • Pengukuran perbedaan antara dua atribut biner berdasarkan pada disimilarity. Misalnya, biner asimetris kesamaan antara objek i dan j dapat dihitung dengan: $$ \operatorname { sim } ( i , j ) = \frac { q } { q + r + s } = 1 - d ( i , j ) $$

    Keterangan:

    • q adalah jumlah atribut yang bernilai 1 untuk kedua objek i dan j

    • r adalah jumlah atribut yang bernilai 1 untuk objek i dan bernilai 0 untuk objek j

    • s adalah jumlah atribut yang bernilai 0 untuk objek i dan bernilai 1 untuk objek j

    • t adalah jumlah atribut yang bernilai 0 untuk kedua objek. Jumlah dari 4 variabel tersebut sama dengan jumlah atribut p, dimana p = q+r+s+t.

    • C. Mengukur Jarak Tipe Data Categorical¶

    1.Overlay Metric¶

    ​ Overlay metric banyak digunakan oleh instance-based learning dan locally weighted learning. Hal ini dapat dilihat jika overlay metric tidak efidien dalam mengukur jarak antara masing-masing pasangan sample, karena gagal memanfaatkan tambahan informasi yang diberikan oleh nilai atribut nominal yang bisa membantu dalam generalisasi. $$ d ( x , y ) = \sum _ { i = 1 } ^ { n } \delta ( a _ { i } ( x ) , a _ { i } ( y ) ) $$

    Keterangan:

    • ai(x) dan ai(y) = nilai atribut pada data ke-i yaitu Ai
    • n= batas jumlah banyaknya data
    2.Value Difference Metric¶

    VDM dikenalkan oleh Standfill and Waltz, versi sederhana dari VDM tanpa skema pembobotan didefinsisikan dengan: $$ d ( x , y ) = \sum _ { i = 1 } ^ { n } \sum _ { c = 1 } ^ { C } \left| P ( c | a _ { i } ( x ) ) - P ( c | a _ { i } ( y ) ) \right | $$ VDM mengasumsikan bahwa dua nilai dari atribut adalah lebih dekat jika memiliki klasifikasi sama. Pendekatan lain berbasi probabilitas adalah SFM (Short and Fukunaga Metric) yang kemudian dikembangkan oleh Myles dan Hand dan didefinisikan dengan: $$ d ( x , y ) = \sum _ { c = 1 } ^ { C } \left | P ( c | x ) - P ( c | y ) \right| $$

    Keterangan:

    • P(c|ai(x))P(c|ai(x)) =probabilitas bersyarat
    • P(c|x) dan P(c|y)=probabilitas keanggotaan kelas
    • n = batas jumlah banyaknya data
    3.Minimum Risk Metric¶

    Ukuran ini dipresentasikan oleh Blanzieri and Ricci, minimum risk metric meminimumkan resiko kelasahan dalam pengklasifikasian suatu sample data yang didefinisikan dengan formula berikut ini: $$ d ( x , y ) = \sum _ { c = 1 } ^ { C } P ( c | x ) ( 1 - P ( c | y ) ) $$

    Keterangan:

    • P(c|x)dan P(c|y)=probabilitas keanggotaan kelas
    • c= batas jumlah banyaknya data

    D. Mengukur Jarak Tipe Ordinal¶

    ​ Ordinal attribute adalah atribut dengan nilai-nilai yang kemungkinan memiliki urutan yang mempunyai arti atau tingkatan(ranking), akan tetapi jarak antara nilai-nilainya tidak diketahui. Ordinal attribute berguna untuk mendaftarkan taksiran suatu kualitas yang tidak bisa diukur secara obyektif. Oleh karena itu ordinal attribute biasanya digunakan dalam survey atau rating. Berikut adalah contoh atribut ordinal:

    1). nilai ujian (A, B, C, D, E)

    2). ranking kelas (1, 2, 3, dst)

    3). kualitas air (sangat baik, baik, cukup, buruk, sangat buruk)

    4). tingkat kepintaran (jenius, pintar, cukup, kurang cukup, sangat kurang)

    5). ukuran baju (small, medium, large, extra large)

    Karena setiap atribut ordinal dapat memiliki jumlah state yang berbeda, diperlukan untuk memetakan rentang setiap atribut ke [0,0, 1.0] sehingga setiap atribut memiliki bobot yang sama dengan menggunakan formula berikut ini: $$ z _ { i f } = \frac { r _ { i f } - 1 } { M _ { f } - 1 } $$

    E. Menghitung Jarak Tipe Campuran¶

    Menghitung ketidaksamaan antara objek dengan atribut campuran yang berupa nominal, biner simetris, biner asimetris, numerik, atau ordinal yang ada pada kebanyakan databasae dapat dinyatakan dengan memproses semua tipe atribut secara bersamaan. Salah satu teknik tersebut menggabungkan atribut yang berbeda ke dalam matriks ketidaksamaan tunggal dan menyatakannya dengan skala interval antar [][0,0,1.0][0,0,1.0]Misalkan data berisi atribut p tipe campuran. Ketidaksamaan (disimilarity ) antara objek i dan j dinyatakan dengan : $$ d ( i , j ) = \frac { \sum _ { f = 1 } ^ { p } \delta _ { i j } ^ { ( f ) } d _ { i j } ^ { ( f ) } } { \sum _ { f = 1 } ^ { p } \delta _ { i j } ^ { ( f ) } } $$

def chordDist(v1,v2,jnis):
    jmlh=0

    #Menghitung Jarak Binary
def binaryDist(v1,v2,jnis):
    q=0
    r=0
    s=0
    t=0
    for x in range (len(jnis)):
        if (int(k.values.tolist()[v1][jnis[x]]))==1 and (int(k.values.tolist()[v2][jnis[x]]))==1:
            q=q+1
        elif (int(k.values.tolist()[v1][jnis[x]]))==1 and (int(k.values.tolist()[v2][jnis[x]]))==2:
            r=r+1
        elif (int(k.values.tolist()[v1][jnis[x]]))==2 and (int(k.values.tolist()[v2][jnis[x]]))==1:
            s=s+1
        else:
            t=t+1
    return ((r+s)/(q+r+s+t))
